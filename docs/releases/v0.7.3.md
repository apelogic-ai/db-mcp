# db-mcp v0.7.3

Release date: 2026-03-19

## Summary

`v0.7.3` hardens the executor-style daemon path by making `prepare_task(...)` much richer and easier for frontier models to use correctly. The daemon now returns full semantic guidance and full schema context in one structured packet, orders the most important decision rules ahead of schema-heavy details, and supports cleaner benchmark isolation through scenario filtering.

## Highlights

- Expanded daemon context so `prepare_task(...)` includes:
  - full domain model
  - full business rules
  - full schema descriptions
  - explicit disambiguation hints
  - adaptive examples and rules
- Reordered daemon context so semantic guidance appears before schema-heavy fields.
- Added `--scenario` filtering to `db-mcp-benchmark run`.
- Removed connection-specific naming from repo code, docs, comments, tests, branch naming, and the active PR path for this work.

## Why this release matters

The daemon path is the executor-style product surface for `db-mcp`. This patch improves the quality of the single prepared context packet without expanding the MCP tool surface beyond:

- `prepare_task(question, connection?, context?)`
- `execute_task(task_id, sql, confirmed=False)`

That keeps the product simple for agent hosts while giving the model more of the information it needs up front.
