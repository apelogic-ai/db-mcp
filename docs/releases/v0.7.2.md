# db-mcp v0.7.2

Release date: 2026-03-18

## Summary

`v0.7.2` is a patch release focused on shipping the first insider-agent control plane behind the existing daemon/runtime surface. It adds an opt-in background curator that can bootstrap semantic artifacts after deterministic onboarding, keeps all serving-path MCP behavior unchanged, and introduces a universal direct-LLM backend alongside a Claude Code-backed insider path.

## Highlights

- Added the insider-agent control plane with staged review, runtime metadata, and background supervision inside the daemon host.
- Added a thin `openai-compatible` direct-model backend using `pydantic-ai`, plus a separate `claude-code` backend for pre-authenticated local agent usage.
- Kept the request path unchanged: insider work stays background-only and operator-governed.

## Breaking changes

- None

## Features

- Added `db-mcp insider` CLI commands for status, events, runs, budget inspection, manual bootstrap, and staged review actions.
- Added a SQLite-backed insider store, transparent `.insider/` vault artifacts, and a shared `InsiderService` boundary for future BICP/UI integration.
- Added automatic `new_connection` bootstrap emission after deterministic onboarding completes.
- Added direct LLM support for OpenAI-compatible endpoints via `provider: "openai-compatible"` with optional `base_url`.
- Added a Claude Code insider backend via `provider: "claude-code"` for environments that already have local authenticated CLI access.

## Fixes

- Removed Anthropic-specific assumptions from the generic direct LLM path so insider configuration can target any OpenAI-compatible endpoint.
- Refactored insider runtime and operator surfaces to share one control-plane service instead of parallel CLI/runtime code paths.
- Added regression coverage for provider selection, config merging, review application, onboarding-triggered bootstrap emission, and supervisor behavior.

## Security

- None

## Upgrade notes

- Insider remains disabled by default. Existing users do not need to change anything unless they want to opt in.
- The generic direct backend now uses `provider: "openai-compatible"` with `api_key_env` and optional `base_url`.
- `provider: "claude-code"` remains available for local authenticated CLI-driven insider runs.

## Known issues

- `uv run pytest tests/ -v` still reports the existing `PytestReturnNotNoneWarning` in `tests/test_database.py::test_connection`.
