# db-mcp v0.8.1

Release date: 2026-03-25

## Summary

`v0.8.1` is a narrow patch release on top of `v0.8.0`. It adds a shipped connector-template
catalog for common API integrations, makes those templates easy to contribute as single-file PRs,
and fixes API connector auth loading so quoted `.env` values work correctly in template-backed
flows.

## Highlights

- Added built-in connector templates for Jira, Dune, and Superset.
- Added CLI support to list shipped templates and initialize API connections directly from a
  template.
- Fixed quoted `.env` value handling in the API connector.

## Breaking changes

- None

## Features

- Added a file-based connector-template catalog under
  `packages/core/src/db_mcp/static/connector_templates/`.
- Added `db-mcp connector templates`.
- Added `db-mcp init <name> --template <template-id>`.
- Extended `db-mcp connector validate` to accept full template files.
- Added documentation for the single-file template contribution workflow.

## Fixes

- Fixed API connector `.env` parsing to strip surrounding quotes before resolving auth values.
- Fixed package ignore rules so shipped connector templates are tracked and included in release
  artifacts.

## Security

- None

## Upgrade notes

- The template catalog is currently CLI-first. The UI wizard still uses the generic connector flow.
- Template contributions that fit the existing connector contract can be added as single YAML files
  without Python changes.
