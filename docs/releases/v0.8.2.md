# db-mcp v0.8.2

Release date: 2026-03-25

## Summary

`v0.8.2` is a focused patch release on top of `v0.8.1`. It adds a shipped Metabase template to the
built-in connector-template catalog and generalizes the API connector so Metabase-style auth,
schema, and SQL-over-HTTP flows fit the generic template path cleanly.

## Highlights

- Added a built-in Metabase template to the shipped connector-template catalog.
- Generalized the API connector to support configurable login-token headers, structured SQL request
  bodies, and schema metadata endpoints.
- Kept existing `type: metabase` connectors working as a backward-compatible wrapper.

## Breaking changes

- None

## Features

- Added `packages/core/src/db_mcp/static/connector_templates/metabase.yaml`.
- Added generic login-token auth support in the API connector, including configurable header names
  and token prefixes.
- Added generic `body_template` support for SQL execution endpoints in the API connector.
- Added generic API schema metadata support via a declared `schema` endpoint.

## Fixes

- Fixed the gap between Metabase runtime support and the shipped template library, so Metabase now
  shows up in `db-mcp connector templates` and `db-mcp init --template ...`.
- Fixed nested columnar API response extraction for Metabase-style `data.cols` / `data.rows`
  payloads.
- Removed incidental provider-specific wording from generic API examples in the CLI/help path.

## Security

- None

## Upgrade notes

- Initialize a Metabase connection with `db-mcp init <name> --template metabase`.
- The shipped Metabase template uses placeholder database id `1`; update the schema endpoint path
  and SQL request body to your real Metabase database id after initialization.
- Existing `type: metabase` connections remain supported.
