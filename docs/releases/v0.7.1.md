# db-mcp v0.7.1

Release date: 2026-03-18

## Summary

`v0.7.1` adds the executor-style daemon MCP path on top of the new runtime core and makes it production-usable for read-heavy analytics workflows. The daemon surface is intentionally small: prepare compact structured context once, execute SQL once, and return grounded results without exposing the older shell/code control flow.

## Highlights

- Added a daemon-first local control plane via `db-mcp up`.
- Added a strict two-tool daemon MCP surface:
  - `prepare_task(question, connection?, context?)`
  - `execute_task(task_id, sql, confirmed=False)`
- Added benchmark coverage for the daemon HTTP MCP path.
- Fixed daemon read-query execution so async backend handles are resolved inline before the answer comes back to the client.

## Why this release matters

This patch is the first release where the executor-style product path is both:

- architecturally distinct from legacy `db-mcp` modes, and
- benchmark-competitive on a hard pack (`top-ledger`) with frontier models.

In the latest `top-ledger` run, the daemon path reached:

- `runtime_daemon`: `10/10`, `20.4s`, `$0.760`

while preserving grounded SQL answers and a strict two-step tool flow over HTTP MCP.
