# db-mcp v0.7.4

Release date: 2026-03-20

## Summary

`v0.7.4` tightens the executor-style daemon path by making the first `prepare_task(...)` response compact and decision-oriented again. Instead of front-loading the full schema and raw semantic artifacts every time, daemon mode now emphasizes high-signal decision hints and focused schema on the first pass, and keeps the broader context available for refinement calls.

## Highlights

- Reworked daemon `prepare_task(...)` so the first pass stays lean and focused on:
  - decision hints
  - disambiguation guidance
  - focused schema
  - targeted examples and rules
- Reserved expanded raw context and full schema for follow-up `prepare_task(..., context=...)` refinement calls.
- Updated daemon prompts and regression coverage to reinforce the intended progressive `prepare_task(...)` then one-shot `execute_task(...)` contract.

## Why this release matters

The executor-style daemon surface should help the model make better choices, not overwhelm it with context. `v0.7.3` proved that always sending the full semantic and schema payload up front could regress simple questions. This patch restores a compact first pass while keeping the broader refinement path available when the agent actually needs more support.
