# db-mcp v0.8.12

## Summary

`v0.8.12` is a patch release on top of `v0.8.11`. It hardens connector runtime loading so
persisted `connector.yaml` files with legacy or extra fields do not break Metabase and API
connections during load, discovery, or save round-trips.

## Highlights

- Fixed runtime loading to ignore unknown top-level `connector.yaml` fields while keeping the
  explicit connector contract validator strict.
- Added Metabase payload normalization so legacy endpoint metadata such as `body_template` and
  endpoint `description` no longer breaks runtime loading or save/reload cycles.
- Added e2e coverage for noisy generic API configs, legacy Metabase configs, and registry-level
  connection loading.

## Breaking changes

- None

## Features

- Runtime connector loading is now more tolerant of persisted config drift in field installations.

## Fixes

- Fixed field-reported Metabase connection failures that continued after `v0.8.11` because saved
  connector YAML still contained legacy endpoint metadata.
- Fixed runtime contract validation so extra top-level `connector.yaml` keys no longer abort
  connector loading when `spec_version` is present.
- Fixed Metabase round-trip persistence so dangerous legacy fields are scrubbed instead of being
  re-emitted into saved connector configs.

## Security

- None

## Upgrade notes

- Upgrade to `v0.8.12` if you have existing saved Metabase or API connections that may contain
  legacy compatibility fields.
- The runtime now tolerates extra top-level connector keys, but `db-mcp connector validate`
  remains strict by design.
