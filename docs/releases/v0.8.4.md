# db-mcp v0.8.4

Release date: 2026-03-26

## Summary

`v0.8.4` is a patch release on top of `v0.8.3`. It fixes the UI wizard transition after API
connection testing so onboarding can continue into the Discover step instead of resetting back to
Connect.

## Highlights

- Fixed the API connection wizard so `Next` after a successful connect test no longer bounces back
  to Connect.
- Made wizard step locking respect the locally saved connection name, not just the URL query
  string.
- Added a regression test for the wizard step-lock helper.

## Breaking changes

- None

## Features

- Added UI test coverage for wizard step locking during the connect-to-discover transition.

## Fixes

- Fixed a client-side race in the connection wizard where the `#discover` transition could be
  rejected before the `name` query param updated.
- Fixed step navigation state so Discover and Sample are unlocked immediately after a connection is
  saved locally.

## Security

- None

## Upgrade notes

- For API onboarding, entering a working naked `base_url`, passing `Test`, and clicking `Next`
  should now reliably land on the Discover step.
- OpenAPI endpoint discovery still runs when you click `Discover` on step 2.
