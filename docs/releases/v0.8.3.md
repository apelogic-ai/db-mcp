# db-mcp v0.8.3

Release date: 2026-03-26

## Summary

`v0.8.3` is a patch release on top of `v0.8.2`. It fixes API onboarding for OpenAPI-backed
services whose naked base URL returns `404`, while still allowing db-mcp to discover the spec,
resolve the actual API base URL, and extract endpoints cleanly.

## Highlights

- Fixed API connect testing so a reachable OpenAPI document is enough to pass onboarding.
- Added direct OpenAPI document URL support to discovery.
- Persisted `spec_url` in `connector.yaml` and reused it during rediscovery.

## Breaking changes

- None

## Features

- Added `spec_url` to the API connector contract and runtime config loader.
- Added OpenAPI/Swagger document URL detection in the discovery pipeline.
- Added spec-derived API base URL resolution from OpenAPI `servers` and Swagger host/basePath.

## Fixes

- Fixed the connection wizard flow for APIs where the base URL returns `404` but a spec exists at
  a well-known path such as `/openapi.json`.
- Fixed direct-spec onboarding so discovery parses the spec instead of falling back to probe mode.
- Fixed persisted connector state so explicit `spec_url` values survive save/load cycles and schema
  export stays in sync.

## Security

- None

## Upgrade notes

- If an API serves its OpenAPI document at a well-known path under the configured base URL, the
  connect step should now pass automatically.
- If the provider serves the spec from a nonstandard location, set `spec_url` explicitly and
  db-mcp will reuse it for discovery and rediscovery.
