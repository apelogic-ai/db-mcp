# db-mcp v0.5.37

## Highlights
- UI SQL connection tests now forward `connect_args` for ad-hoc database URLs.
- Ad-hoc URL tests also parse `http_scheme` and `verify` from the URL query string.
- SQL introspection honors `connect_args` across catalogs, schemas, tables, columns, and samples.

## Fixes
- UI “Database URL” tests no longer default to HTTPS for Trino when HTTP is required.
- BICP SQL connection tests accept explicit `connect_args` or URL param fallbacks.

## Upgrade Notes
- Keep `DATABASE_URL` secrets in `.env`; keep `connect_args` in `connector.yaml`.
- For ad-hoc testing, you may pass `connectArgs` in the UI; `?http_scheme=http&verify=false` still works.
