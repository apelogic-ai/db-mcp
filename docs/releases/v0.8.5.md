# db-mcp v0.8.5

Release date: 2026-03-26

## Summary

`v0.8.5` is a patch release on top of `v0.8.4`. It fixes OpenAPI endpoint discovery for APIs that
use required contextual path parameters in collection routes and for OpenAPI 3.1 specs that
express nullable fields with type arrays.

## Highlights

- Fixed OpenAPI discovery so contextual collection routes like `/tenants/{tenant}/articles` are
  treated as real collection endpoints.
- Fixed field extraction for OpenAPI 3.1 nullable union types such as `["string", "null"]`.
- Verified the fix against the Boost public API spec, which now discovers `31 endpoints`.

## Breaking changes

- None

## Features

- Added discovery regressions for contextual collection paths and OpenAPI 3.1 nullable type
  arrays.

## Fixes

- Fixed the path filter so only likely detail endpoints ending in a path param are skipped.
- Fixed field-type normalization so OpenAPI 3.1 type arrays no longer crash spec parsing.
- Fixed real API onboarding for the Boost public API, which now resolves via OpenAPI discovery and
  returns discovered endpoints.

## Security

- None

## Upgrade notes

- APIs that scope collections under required context params should now discover endpoints normally.
- The Boost API example from onboarding now resolves with `strategy=openapi` and `31 endpoints`.
