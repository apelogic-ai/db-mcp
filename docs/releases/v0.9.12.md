# Release v0.9.12 - Install lifecycle commands & UI static-mount fix

## Overview

Adds two long-missing CLI lifecycle commands — `db-mcp uninstall` and `db-mcp update` — and fixes a regression in v0.9.11 binaries that caused the web dashboard (`db-mcp ui`) to silently 404 every page when the bundled UI assets weren't available at app construction time.

## Highlights

- New `db-mcp update` — checks GitHub for the latest release and installs in place. Supports `--check`, `--version X.Y.Z`, and `-y`.
- New `db-mcp uninstall` — removes the binary; `--purge` (behind a second confirmation) wipes `~/.db-mcp`.
- Bug fix: `db-mcp ui` no longer returns `{"detail":"Not Found"}` for every dashboard page when the PyInstaller static bundle's directory check fails at startup.

## New Features

- **`db-mcp update`**: queries `api.github.com/repos/apelogic-ai/db-mcp/releases/latest`, prints current vs. latest, and reinstalls in place. Mirrors `scripts/install.sh`'s macOS cache symlink layout so subsequent uninstalls work cleanly. `--check` reports without installing; `--version` pins a specific release; `-y` skips the install confirmation.
- **`db-mcp uninstall`**: removes the binary at the resolved install path (and the cached versioned binary on macOS when symlinked). `--purge` additionally deletes `~/.db-mcp` (config, connections, knowledge vault, cached binaries) behind a second explicit confirmation. `-y` skips both prompts for scripted use.
- **CLI help layout**: new "Install" subsection under "General" surfaces `update` and `uninstall` in `db-mcp --help`.

## Bug Fixes

- **UI dashboard 404 regression**: removed the silent-fail `if STATIC_DIR.exists()` guard in `ui_server.py` that caused the StaticFiles mount to never register when the bundle's static directory wasn't usable at `create_app()` time. Now uses `StaticFiles(check_dir=False)` so the mount registers unconditionally; individual file requests cleanly 404 if the directory is genuinely missing. Adds a one-line diagnostic log so the static-dir state is always visible in `~/.db-mcp/ui-server.log`.
